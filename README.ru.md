# Descript.ion Fixer

Утилита для управления и объединения файлов `descript.ion` в стиле файлового менеджера Total Commander.

## Что такое descript.ion?

`descript.ion` — файл, используемый Total Commander и другими файловыми менеджерами для хранения описаний/комментариев к файлам и папкам. Каждая строка содержит имя файла и его описание.

## Возможности

- **Объединение конфликтующих файлов описаний**: Объединяет файлы `descript.*.ion`, возникшие в результате конфликтов синхронизации облачных хранилищ (OneDrive, Dropbox, Google Drive и др.)
- **Удаление устаревших записей**: Удаляет описания для файлов, которые больше не существуют
- **Рекурсивная обработка**: Опциональная обработка подкаталогов рекурсивно
- **Умное разрешение конфликтов**: При объединении приоритет имеют более новые файлы (по времени модификации)
- **Поддержка скрытых файлов**: Корректно обрабатывает скрытые файлы `descript.ion` в Windows
- **Логирование**: Все операции записываются в лог-файл во временной директории системы

## Установка

### Из релиза

Скачайте последний релиз со страницы [Releases](https://github.com/alezhu/descript.ion_fixer/releases):

- `descript.ion_fixer-vX.X.X-windows-x64.zip` — исполняемый файл в ZIP-архиве
- `descript.ion_fixer-vX.X.X-installer.exe` — установщик Windows (64-бит)

### Сборка из исходного кода

Требования:
- Zig 0.15.2 или новее

```bash
git clone https://github.com/alezhu/descript.ion_fixer.git
cd descript.ion_fixer
zig build -Doptimize=ReleaseSmall
```

Исполняемый файл будет создан в `zig-out/bin/descript.ion_fixer.exe`.

## Использование

```bash
descript.ion_fixer.exe <путь_к_папке> [--recursive|-r]
```

### Аргументы

| Аргумент | Описание |
|----------|----------|
| `<путь_к_папке>` | Путь к обрабатываемой директории (обязательно) |
| `--recursive`, `-r` | Обрабатывать подкаталоги рекурсивно (опционально) |

### Примеры

Обработка одной директории:
```bash
descript.ion_fixer.exe "C:\Мои файлы"
```

Обработка директории и всех подкаталогов:
```bash
descript.ion_fixer.exe "C:\Мои файлы" --recursive
```

Краткая форма:
```bash
descript.ion_fixer.exe "C:\Мои файлы" -r
```

## Как это работает

1. **Сканирование директории**: Утилита сканирует указанную директорию и собирает:
   - Все существующие файлы
   - Все файлы `descript.*.ion` (например, `descript.ion`, `descript_backup.ion`)
   - Все подкаталоги (если включён рекурсивный режим)

2. **Загрузка основного описания**: Загружает `descript.ion` и удаляет записи для файлов, которые больше не существуют

3. **Объединение дополнительных файлов**: Для каждого файла `descript.*.ion`:
   - Читает все пары ключ-значение
   - Добавляет новые записи в основной `descript.ion`
   - Обновляет существующие записи, только если исходный файл новее
   - Удаляет обработанный файл `descript.*.ion`

4. **Сохранение результата**: Сохраняет объединённый файл `descript.ion` с сохранением атрибута hidden

5. **Обработка подкаталогов**: Если указан `--recursive`, повторяет процесс для всех подкаталогов

## Формат файла

Формат файла `descript.ion`:
```
filename.txt Описание для этого файла
"файл с пробелами.txt" Описание для файла с пробелами
another_file.dat
```

- Файлы без пробелов: `имя_файла описание`
- Файлы с пробелами: `"файл с пробелами" описание`
- Пустые описания разрешены

## Логирование

Все операции записываются в `%TEMP%\description_fixer.log`. Это полезно для:
- Отладки проблем
- Просмотра внесённых изменений
- Аудита
